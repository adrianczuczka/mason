# Mason

A context engineering CLI that generates intelligent CLAUDE.md files through structured codebase analysis.

Instead of manually writing agent instructions or asking an LLM to glance at your repo, Mason uses AST parsing, git history mining, and dependency graph analysis to find patterns you wouldn't think to look for — then turns them into actionable rules.

## Install

```bash
npm install -g mason-context
```

Or run directly:

```bash
npx mason-context init
```

## Usage

### `mason init [dir]`

Analyze your codebase, ask clarifying questions, and generate a CLAUDE.md file.

```bash
mason init                  # current directory
mason init ./my-project     # specific directory
mason init --no-questions   # skip interactive prompts
```

### `mason analyze [dir]`

Run all analyzers and print findings without generating anything.

```bash
mason analyze
```

### `mason update [dir]`

Re-analyze and update an existing CLAUDE.md. Only asks about new findings since the last run.

```bash
mason update
mason update --no-questions
```

## What it detects

### Git History
- Commit message conventions (conventional commits, ticket references)
- Stale directories that haven't been touched in months
- Frequently changed files (high conflict risk)

### Import Conventions
- Path alias usage vs relative imports
- Barrel file (index.ts) patterns
- `import type` usage
- Import ordering (third-party first vs local first)

### Test Conventions
- Test framework detection (Vitest, Jest, Mocha, Playwright, Cypress)
- Test file locations (colocated vs separated)
- Test patterns (describe/it vs flat test())
- Coverage configuration

### Code Patterns (AST)
- Export style (named vs default)
- Naming conventions (camelCase functions, UPPER_SNAKE constants)
- Error handling (try/catch vs .catch())
- Async patterns (async/await vs .then())
- React patterns (functional vs class, arrow vs declaration, forwardRef)

### Dependency Graph
- Monorepo structure and package boundaries
- Circular dependency detection
- Isolated/orphaned packages
- Misplaced dependencies (dev tools in production deps)

## Example output

```markdown
# CLAUDE.md
<!-- Generated by mason-context. Last updated: 2026-04-07 -->

## Code Conventions

- Use conventional commit format: type(scope): description
- Prefer named exports over default exports
- Use async/await instead of .then() chains
- Use `import type` for type-only imports

## Architecture

- This is a monorepo with 3 packages. Respect package boundaries.

## Areas of Note

- Directory "legacy/" hasn't been modified in 14 months — do not refactor unless asked.
```

## How it works

Mason runs 5 analyzers in parallel, each producing **findings** (evidence-backed observations with confidence scores) and **gaps** (things it couldn't determine). Gaps become interactive questions. Findings above a 60% confidence threshold become rules in your CLAUDE.md.

A cache file (`.mason-cache.json`) tracks previous findings and answers, so `mason update` only asks about what's changed.
